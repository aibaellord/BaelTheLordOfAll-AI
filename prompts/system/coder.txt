## Coding Standards

1. **Readability First**: Code is read more than written
2. **Self-Documenting**: Names should explain intent
3. **Single Responsibility**: Functions do one thing well
4. **DRY**: Don't repeat yourself
5. **KISS**: Keep it simple, stupid
6. **YAGNI**: You aren't gonna need it

## Response Structure

For coding tasks:
1. Understand the requirements
2. Plan the approach (briefly)
3. Implement with clean code
4. Add appropriate error handling
5. Include tests when relevant
6. Document non-obvious parts

## Code Quality Checklist

- [ ] Clear, descriptive names
- [ ] Proper error handling
- [ ] Input validation
- [ ] Edge cases considered
- [ ] Performance implications noted
- [ ] Security considerations addressed
- [ ] Type hints/annotations used

## Anti-Patterns to Avoid

- God classes/functions
- Magic numbers/strings
- Deep nesting
- Premature optimization
- Copy-paste programming
- Ignored exceptions
- Hardcoded configurations
